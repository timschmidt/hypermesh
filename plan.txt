hypermesh revision plan
=======================

Goal
----

Make hypermesh a minimal, full-featured, exact, high-performance, provenance-preserving mesh kernel. `ExactMesh` remains the primary top-level type. The crate should contain geometry/fact storage, acceleration structures, exact graph/arrangement/cell algorithms, and useful kernel errors. csgrs should own workspace-level named boolean APIs, operation policy, handoff routing, adapters, and product-facing reports.

Target public surface
---------------------

1. Primary owned type:
   - `ExactMesh`: exact vertices, triangles, retained validation facts, exact bounds, construction provenance, and cached predicate/fact evidence.
   - Checked constructors from exact coordinates, integers, and explicit exact source records.
   - Minimal lossy input only as named adapter constructors, not the main path.

2. Borrowed APIs:
   - `ExactMeshRef<'a>` / `MeshView<'a>` for vertices, triangles, facts, bounds, adjacency, face planes, and validation replay.
   - `FaceRef`, `TriangleRef`, `EdgeRef`, and borrowed graph/arrangement views for algorithms that should not clone meshes.
   - Prepared/cached borrowed broad-phase views where repeated mesh-pair queries benefit.

3. Kernel modules:
   - Facts and validation.
   - Exact AABB/broad phase.
   - Narrow triangle predicates and retained constructions.
   - Intersection graph event extraction.
   - 2D coplanar arrangements and 3D arrangement/cell complexes.
   - Exact simplification, loop triangulation, CDT face cells, winding, volumetric-cell evidence.
   - Specialized exact shortcuts as internal algorithms, not policy.

4. Kernel errors:
   - `ExactMeshError` / `ExactMeshBlocker` should identify invalid indices, degeneracy, non-manifold topology, undecidable predicate, failed exact construction, stale fact replay, unsupported cell materializer, and policy-required-but-missing evidence.
   - Every error should carry source side, face/edge/vertex indices, predicate/construction family, and provenance where possible.

Public API removals or moves
----------------------------

Remove:

- `ExactBooleanWorkspace`.
- `ExactBooleanRequest`, `ExactBooleanOperation` as public operation API, `ExactBooleanEvaluation`, `ExactBooleanResult`.
- Boundary/regularization/lower-dimensional/unresolved policies.
- Public report families in `src/reports.rs`.
- Handoff packages, readiness summaries, proposal acceptance reports, artifact routing vocabulary, and consumer-domain APIs.
- Bevy/demo/runtime/export examples that depend on product-level semantics.

Keep in hypermesh but narrow:

- Low-level graph/arrangement/cell-complex builders and replay validation.
- Certified shortcut algorithms for adjacent, contained-adjacent, convex, affine/orthogonal, and box solids.
- Optional triangulation helpers if they are used as exact kernel algorithms.
- Exact validation/audit facts only when they are about `ExactMesh` itself.

Performance architecture
------------------------

Performance must be designed around Yap-style retained facts from the first API
shape, not added after the boolean pipeline exists. The kernel should make every
expensive stage consume a cacheable fact package with source-replay validation.

1. Mesh-local fact caches.
   - Store exact vertex facts, edge facts, triangle facts, determinant-form face
     planes, exact face AABBs, optional support/k-DOP intervals, adjacency, edge
     incidence, orientation, manifold/boundary summaries, and provenance in
     `ExactMesh`.
   - Prefer compact stable indices and borrowed views over cloning facts into
     result objects.
   - Keep exact rational keys as bucket accelerators only; equality and topology
     still replay through exact predicates when keys are absent or symbolic.

2. Mesh-pair session caches.
   - Expose low-level borrowed caches for pair broad phase, retained face-pair
     classification, intersection graph events, coplanar overlap graphs, split
     topology plans, local arrangements, winding/ownership facts, and
     triangulation products.
   - Invalidate by source identity plus construction version rather than by
     object address alone.
   - Make cache freshness explicit: current, replay-blocked, stale, or
     unsupported with a typed blocker.

3. Stage metrics.
   - Bench candidate-pair count, AABB/k-DOP rejection rate, predicate filter hit
     rate, constructed-point count, indirect-predicate use count, local
     arrangement cell count, CDT triangle count, winding query count, replay
     validation cost, cache hit rate, memory bytes per retained fact, and
     parallel task count.
   - Keep these metrics in benchmark/debug surfaces, not default public policy.

Refactor phases
---------------

1. Freeze source-compatible facts.
   - Define the minimal `ExactMesh` public contract.
   - Add borrowed view types without changing existing internals.
   - Add compatibility shims for current tests.

2. Separate workspace orchestration.
   - Extract `src/workspace.rs`, public request/evaluation/materialization flow, and most of `src/reports.rs` into csgrs.
   - Keep graph, arrangement, attempt internals callable from csgrs through low-level borrowed APIs.

3. Split policy from algorithms.
   - Move named boolean operation, validation-output policy, regularization semantics, boundary behavior, and consumer readiness to csgrs.
   - Keep only kernel-local blockers and exact construction choices.

4. Rework `src/boolean.rs`.
   - Pull out reusable kernel algorithms into smaller modules: graph preflight, arrangement attempt, cell selection, shortcut certification, output triangulation.
   - Leave a thin internal algorithm layer, not public orchestration.
   - Move large test-only helper sections into tests.

5. Preserve and improve performance structures.
   - Keep exact AABB sweep as the baseline broad phase; add a pluggable exact
     broad-phase trait that can support Morton tiling, BVH, k-DOP/support-DOP
     intervals, or octree cells without changing narrow-phase outputs.
   - Implement EMBER-style adaptive local arrangements for one-shot mesh
     booleans: recursively subdivide space, propagate local winding reference
     facts, split only local leaf polygons, and terminate early when a leaf
     cannot contribute to the requested result.
   - Do not prebuild a global arrangement unless the caller requests reusable
     multi-operation evidence or the local path detects that global replay will
     be cheaper.
   - Use indirect predicates for constructed intersection objects where eager
     exact coordinates are expensive. Retain the source primitives and
     construction family for line-plane, plane-plane-plane, segment-plane,
     projected edge-edge, and radial-order predicates.
   - Keep exact co-refinement, radial sorting around non-manifold intersection
     edges, and CDT face remeshing as the fallback complete path for difficult
     general inputs.
   - Keep CDT face-cell triangulation and local arrangement decomposition as
     core. Use symbolic perturbation or an equivalent retained tie-breaker for
     co-circular or duplicate projected configurations.
   - Keep specialized exact shortcuts for disjoint, adjacent, contained,
     convex, affine/orthogonal, and box solids, but require them to return the
     same retained blockers/facts as the generic path.

6. Tighten provenance and useful errors.
   - Require every graph event, split point, arrangement vertex/edge/cell, simplified cell, and output triangle to retain source provenance or construction evidence.
   - Make unsupported cases explicit blockers such as `NeedsCoplanarVolumetricCells`, not generic unsupported operation diagnostics.
   - Validate caches by source replay before they authorize topology decisions.

7. Tests and docs.
   - Split `tests/exact_mesh.rs` into kernel-focused integration tests and csgrs policy tests.
   - Retarget fuzz harnesses to exact mesh construction, arrangements, and artifact adapters according to ownership.
   - Rewrite README around `ExactMesh`, borrowed views, exact facts, low-level algorithms, and csgrs-level booleans.

Research integration
--------------------

Primary research conclusions to bake into the plan:

- EMBER is the main performance target for one-shot exact booleans. Use
  adaptive recursive spatial subdivision, local per-leaf BSP/arrangement
  splitting, propagated local winding number references, exact plane-based
  representation, early-out leaf pruning, and a work-stealing parallel task
  model. Source: https://graphics.rwth-aachen.de/media/papers/339/ember_exact_mesh_booleans_via_efficient_and_robust_local_arrangements.pdf
- Mesh Arrangements for Solid Geometry is the robust semantic fallback:
  construct exact arrangements, assign winding-number vectors to cells, and let
  csgrs extract named boolean results from those vectors. It is not the default
  fastest path, but it is the correctness model for arbitrary inputs. Source:
  https://www.cs.columbia.edu/cg/mesh-arrangements/
- Levy 2024 should guide the complete exact CSG backend: exact co-refinement,
  exact Weiler/arrangement model, radial sorting around non-manifold
  intersection edges, simple combinatorial handoff between stages, exact
  expansion or multiprecision kernels, and CDT remeshing with retained
  tie-breakers for ambiguous projected cases. Source:
  https://arxiv.org/abs/2405.12949
- Attene indirect predicates should be a core optimization for constructed
  intersections. Represent constructed points by source primitives and
  denominator/numerator facts, then evaluate orientation/incidence/radial-order
  predicates without eagerly materializing every exact coordinate. Source:
  https://arxiv.org/abs/2105.09772
- Interactive and Robust Mesh Booleans validates that exact predicate pipelines
  can be interactive when intermediate facts are lightweight and filtered; use
  it as a performance bar for batch and million-triangle cases. Source:
  https://arxiv.org/abs/2205.14151
- Octree-embedded BSPs are the specialized backend for iterated CSG and
  repeated plane cuts. Keep this separate from the triangle-arrangement kernel,
  but design `ExactMesh` facts so plane-based cells can borrow face-plane and
  provenance caches. Source: https://arxiv.org/abs/2103.02486
- CGAL corefinement/autorefinement is a practical baseline for triangle soup
  refinement and intersection polylines; it also demonstrates where concurrency
  and visitor hooks are useful. Treat snap rounding as an adapter policy, not a
  kernel topology decision. Source:
  https://doc.cgal.org/latest/PMP_Boolean_operations/group__PMP__corefinement__grp.html
- CGAL Nef polyhedra are the semantic reference for closure under boolean,
  complement, boundary, closure, interior, and regularization operations.
  Keep these set-operation semantics in csgrs policy while hypermesh retains
  lower-dimensional artifacts and cell facts. Source:
  https://doc.cgal.org/latest/Nef_3/index.html
- Guigue/Devillers, Moller, and Shewchuk remain narrow-phase/predicate
  baselines: plane-side rejection, exact orientation predicates, and adaptive
  filters should decide branch conditions before any topology mutation. Source:
  https://www.cs.cmu.edu/~quake/robust.html
- Broad phase should evolve from exact AABB sweep to a measured hierarchy:
  Morton tiles for locality, BVH/k-DOP/support-DOP intervals for tighter
  rejection, and octree cells when local arrangements or iterated CSG need
  spatial subdivision. Each broad-phase object is an acceleration fact, never a
  topology certificate.

Definition of done
------------------

- `ExactMesh` is the only primary top-level type documented in hypermesh.
- The default crate root no longer exports workspace-level boolean policy, handoff packages, readiness, proposal, or broad public report APIs.
- Borrowed mesh/face/triangle/arrangement views exist and are preferred for queries.
- Exact facts and acceleration structures replay against source objects before use.
- csgrs can still perform named booleans by ExactMesh.union(ExactMesh); or difference, intersection, xor, transform, inverse, or the borrowed equivalents and all other functions required by the CSG trait.
- Every exact algorithm returns a decided value or a typed useful blocker with retained provenance.
